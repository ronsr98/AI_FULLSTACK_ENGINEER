# Input (Controlled Components)

Controlled inputs (`npm install` then `npm run dev`).

- **Exercise 1** - two controlled inputs (`name`, `age`) held in one state object; the button alerts a message built from both, read straight from state.
- **Exercise 2** - controlled text input + a `select` dropdown. Choosing a fruit updates state and logs "`<name>` selected `<fruit>`" - because the setter is async, the chosen value comes from the event while `name` comes from state.
